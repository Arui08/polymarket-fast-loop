## ResearchWang 帖子学习记录
- 来源：`https://x.com/researchwang/status/2030151219998793775?s=46`
- 可确认片段显示，这是一条关于 **OpenClaw Skills / 配置分层** 的经验帖。
- 已确认信息：
  - 作者强调：当 OpenClaw 版本高于 `3.2` 时，需要在 `openclaw.json` 中把 `tools` 权限设为 `full`，否则智能体容易退化成“只会聊天”。
  - 作者明确：`IDENTITY.md` 的职责是“对外的身份信息”。
- 形成学习：
  1. 遇到“只会聊天不会执行”的问题，先排查工具权限，而不是先怀疑模型或提示词。
  2. 文件职责继续严格分层：`IDENTITY.md` 管身份，`SOUL.md` 管行为风格，`USER.md` 管用户画像，`MEMORY.md` 管长期记忆，`memory/` 管每日上下文。
  3. 后续研究 OpenClaw 技巧时，优先关注 Skills 设计、触发描述、权限边界、分层降噪。
